# SEC 642


بهترین چک‌لیست‌های دوره SEC642
چک‌لیست OWASP Top 10:
بررسی آسیب‌پذیری‌های رایج مثل SQL Injection، XSS، CSRF، Broken Access Control، Security Misconfiguration، etc.

چک‌لیست API Security:
بررسی صحت احراز هویت توکن‌ها، مدیریت نرخ درخواست، کنترل دسترسی دقیق روی endpointها، اعتبارسنجی ورودی JSON و XML.

چک‌لیست Session Management و Authentication:
کنترل کوکی‌ها، بررسی Secure & HttpOnly بودن کوکی‌ها، تست Session Fixation و Session Expiry.

چک‌لیست WAF Bypass:
روش‌های تغییر Payload و رمزگذاری برای عبور از فایروال‌های برنامه تحت وب.

فازهای Recon و Enumeration:
بررسی DNS، Subdomains، دایرکتوری‌ها و فایل‌های مخفی با ابزار wfuzz، dirb و Burp Suite.

چک‌لیست POST/GET Parameter Analysis:
شناسایی پارامترهای ورودی حساس برای تزریق و تغییر داده.

بهترین پیلودهای مورد استفاده در SEC642
پیلودهای SQL Injection:
```
Payloadهای ساده مثل ' OR 1=1 --

Blind SQLi payloads مثل WAITFOR DELAY '00:00:05'

UNION SELECT و استخراج اطلاعات ستون‌ها.
```
پیلودهای XSS:
```
اسکریپت‌های ساده: <script>alert(1)</script>

Obfuscated XSS: <scr<script>ipt>alert(1)</scr</script>ipt>
```
Payload های DOM-XSS و حملات ترکیبی.

پیلودهای Command Injection:

استفاده از ; ls -la یا | cat /etc/passwd

تزریقات خطرناک در پارامترهای ورودی.

پیلودهای SSRF و XML External Entity (XXE):

URLهای Burp Collaborator.

XML با Entity تعریف‌شده برای بررسی fetch داده‌های حساس.

پیلودهای CSRF:

حذف یا جعل توکن CSRF در درخواست‌ها.

پیلودهای فیلترهای WAF Bypass:

کدگذاری URL

استفاده از Unicode و Encodingهای مختلف

شکستن کلیدواژه‌ها به صورت ناقص.

پارامترها و Endpointهای مهم
پارامترهای ورودی GET و POST حساس مثل:
```
id, user, token, username, password, email, redirect
```
پارامترهای مدیریت نشست: sessionid, auth, jwt

Endpointهای کاربردی:
```
/login, /logout, /register

/api/auth, /api/v1/users, /api/v1/data

/admin, /dashboard

/upload, /files, /download

APIهای REST و SOAP:
```
ورودی JSON با فیلدهایی مثل action, resource_id, data

WebSocket endpoints جهت بررسی حملات real-time.

فصل 1: مقدمه و اصول تست نفوذ وب پیشرفته
معرفی روش‌های تست نفوذ جامع و مدرن

مرور چرخه عمر تست نفوذ و فازهای کشف، اکسپلویت و گزارش‌دهی

اهمیت شناخت معماری برنامه و تکنولوژی‌های استفاده شده

نکته عملی: همیشه ابتدا reconnaissance دقیق انجام دهید، همه ورودی‌ها و نقاط تعامل با کاربر را شناسایی کنید.

---

فصل 2: ابزارها و روش‌های reconnaissance پیشرفته
استفاده از ابزارهایی مانند Burp Suite، Nmap، Dirbuster و غیره

تکنیک‌های شناسایی فایل‌ها و دایرکتوری‌های مخفی

تکنیک‌های فنی fingerprinting برای شناسایی سرورها و فریم‌ورک‌ها

نکته عملی: تست‌های crawling دقیق انجام دهید و تمام URLها و پارامترها را لیست کنید.

---

فصل 3: آسیب‌پذیری‌های تزریق (Injection) پیشرفته
SQL Injection، Command Injection، LDAP و تزریقات NoSQL

استفاده از payloadهای پیشرفته و تکنیک‌های Blind Injection

نکته عملی: بهره‌گیری از Burp Intruder و Auto Scanner برای تست خودکار تزریقات.

---

فصل 4: حملات Cross-Site Scripting (XSS) و Cross-Site Request Forgery (CSRF)
بررسی انواع XSS شامل Stored, Reflected و DOM-based

روش‌های ترکیبی و پیشرفته بای‌پس فیلترهای فایروال، و بهره‌برداری از CSRF به همراه XSS

نکته عملی: برای تست XSS از payloadهای Obfuscated و کدهای ترکیبی استفاده کنید.

---

فصل 5: آسیب‌پذیری‌های رمزنگاری و JWT
ضعف‌های متداول در مدیریت Session و کوکی‌ها

حملات روی الگوریتم‌های رمزنگاری نامناسب و حملات Padding Oracle

چگونگی بررسی و سوء استفاده از JWT با حملاتی مانند تغییر الگوریتم، حذف امضا و Key Confusion

نکته عملی: برای بررسی JWT از ابزارهایی مثل jwt.io و Burp Suite plugin استفاده کنید.

---

فصل 6: تست نفوذ API و Web Services
تفاوت‌ها و آسیب‌پذیری‌های رایج SOAP, REST و WebSockets

تست صحت اعتبارسنجی داده‌ها و مکانیزم احراز هویت API

نکته عملی: فیلدهای ورودی API را با JSONها و XMLهای تغییریافته تست کنید.

---

فصل 7: عبور از فایروال‌های وب (WAF)
شناسایی انواع فایروال و نحوه عملکرد آنها

تکنیک‌های تغییر Payload، رمزگذاری و استفاده از کدهای مخفی برای عبور از WAF

نکته عملی: استفاده از ناقص سازی payload و encodingهای متفاوت برای دور زدن فیلترها

---

فصل 8: حملات ترکیبی و بهره‌برداری پیشرفته
ترکیب آسیب‌پذیری‌ها مثل XSS+CSRF، SSRF+Injection و روش‌های chaining حملات

اهمیت فهم جریان‌های کاری در اپلیکیشن برای یافتن نقاط ضعف

نکته عملی: طراحی سناریوهای حمله چندمرحله‌ای و استفاده از Burp Suite برای مدیریت تست‌ها

---

فصل 9: گزارش‌دهی و ارائه نتایج
اصول نگارش گزارش دقیق، جامع، و قابل فهم برای توسعه‌دهندگان

تاکید بر ارائه شواهد، توصیه‌های رفع مشکل و اولویت‌بندی آسیب‌پذیری‌ها

نکته عملی: استفاده از قالب‌های استاندارد گزارش‌نویسی مثل OWASP و SANS برای گزارش‌ها

🛠 ابزارها و تکنیک‌های مورد استفاده

Burp Suite Pro: برای تست آسیب‌پذیری‌های وب و تحلیل ترافیک.

OWASP ZAP: ابزار متن‌باز برای تست امنیت وب‌اپلیکیشن‌ها.
